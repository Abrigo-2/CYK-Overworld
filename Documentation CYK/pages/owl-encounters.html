- Optimizing for 1 Encounter



    I went over the trouble of clearing out needless chunks of code and rearranging the library, to then realize that the "Optimized" version of the library has no noticeable effect in lowering the delay at the start of the mod. Huh.

    At least it's a bit more straightforward to work with...?


    For this demonstration, we'll assume you want to:

    • Play a little cutscene before the Encounter starts.
    • Have Textboxes pop up in said cutscene.
    • Start an encounter.
    • Play a little cutscene after the Encounter ends.


    Setting up the Encounter

    We'll move to the Encounter's script for this.


    The excerpt from the demonstration example explains it well enough: 

    optimizedForOneEncounter = true  -- Loads and builds certain things as soon as the mod starts. Make sure you've ticked this as true!

battleLastLoaded = "Encounter-Only" -- Name of the last loaded Battle file. We make sure it matches the currently loaded battleFile to reduce loading time.
battleFile = (require "Battles/Encounter-Only")()

function EncounterStarting()
    State("NONE")

    -- Add the Overworld here...
    Overworld = (require "OverworldOptimized/OverworldCore")()

    -- Since we're optimizing, you'll have to manually set the avatar data.
    -- Avatars store Player data after an encounter.
    Overworld.CreateAllAvatars_Optimized({"OWKris","OWRalsei","OWSusie","OWGentle",})

    Overworld.SetAvatarProperties_Optimized("OWKris",   90, "Kris")
    Overworld.SetAvatarProperties_Optimized("OWRalsei", 70, "Ralsei")
    Overworld.allAvatars["OWRalsei"].hp = 45
    Overworld.SetAvatarProperties_Optimized("OWSusie",  110, "Susie")
    Overworld.SetAvatarProperties_Optimized("OWGentle", 123, "Gentle")

    
    -- Lets Overworld know which players to track.
    local defaultParty = {"OWKris", "OWRalsei", "OWSusie"}
    for i=1, 3 do  -- Only 3, Deltarune isn't designed for more.
        Overworld.party[i] = Overworld.allAvatars[defaultParty[i]]
    end
    Overworld.GeneratePartyBattleNames()

    -- Add the items, one by one. They don't need to be in the inventory.
    Inventory.AddCustomItem("Dark Candy", "Heals 40HP", 0, "Player")
    Inventory.AddCustomItem("Dark Burger", "Burnt as hell 70HP", 0, "Player")
    Inventory.AddCustomItem("Bandage", "Apt healing", 0, "Player")

    -- Rather than using Inventory.SetInventory, you'll have to use:
    OWinventory = {"Dark Candy", "Dark Candy", "Dark Burger", "Bandage"} 


    Misc.MoveCameraTo(0, 0) -- Place the camera somewhere.
    -- On your cutscene, before you switch to the "INTRO" state,
    -- ensure the camera is in the same position as when you started the encounter.


    -- This here loads the battle.
    Overworld.StartBattleIntro("Encounter-Only", false)

    -- By default, Overworld.CutsceneObj will start a cutscene if it detects this value as true.
    Overworld.CutsceneObj.startSpecialCustscene[1] = true
end

    Setting up the Intro Cutscene

    We'll move to the Overworld/Cutscenes's script for this.

    function EncounterOnlyPreamb()
        -- Freezes playback of the animation if a Textbox is currently being shown.
        if Overworld.TextBox.isActive then return end

        if self.frame == 1 then
            --Misc.MoveCameraTo(0, 0)  -- Move the camera, if you'd like.

            -- Generate the "Actors", aka, sprites that'll move around and act during the scene.
            self.actor[1] = CreateSprite("Overworld/Kris/Idle/2", "Entity")
            self.actor[1].SetPivot(0.5, 0)
            self.actor[1].MoveTo(playerpositions[1][1]*2 - 15, playerpositions[1][2])

            self.actor[2] = CreateSprite("Overworld/Ralsei/Idle/2", "Entity")
            self.actor[2].SetPivot(0.5, 0)
            self.actor[2].MoveTo(playerpositions[2][1]*2 - 8, playerpositions[2][2] + 8)

            self.actor[3] = CreateSprite("Overworld/Susie/Idle/2", "Entity")
            self.actor[3].SetPivot(0.5, 0)
            self.actor[3].MoveTo(62*2 - 5, playerpositions[3][2] + 4 )
    
        elseif self.frame == 25 then
            --self.isTextboxAbove = false  
            self.Textbox("scene1") -- Call code like this as many time as you'd like.
            -- Check out the Cutscenes page, as well as the Overworld/Cutscenes file
            -- for ways to better structure longer scenes.
        
        elseif self.frame > 61 then
            -- Hides the actors, rather than removing them.
            -- This is so we can reuse them on the Outro cutscene.
            for i=1, #self.actor do
                self.actor[i].alpha = 0
            end

            self.currentCutscene   = nil -- This stops the cutscene from playing.

            State("INTRO") -- This starts the encounter.
            Audio.PlaySound("battlestart") -- This manually play the sound. 
            -- It'll be ommited, since skipintro is set true for the demonstration.
        end

        self.frame = self.frame + 1

    As stated above, check out the Cutscenes page, as well as the Overworld/Cutscenes script for tools to better structure the code.



    Since we're optimizing for 1 Encounter, the Textbox will load its dialogues from the JSON Overworld/AllDialogues file.
    More on this can be found on the respective Dialogues section.


    Setting up the Outro Cutscene



    There are two methods for triggering the Outro cutscene:



    Method 1: The EnteringState() call within the Encounter's script.

    function EnteringState(newstate, oldstate)
    if oldstate == "BEFOREDONE" then
        ----------
        CYK.State("NONE")
        CYK.EncunterWrapUp()

        Overworld.CutsceneObj.startSpecialCustscene[2] = true
        ----------
    end
    battleFile.EnteringState(newstate, oldstate)
end

    Method 2: The OnEncounterEnding() call within the Overworld/RoomHandler's script.

    function self.OnEncounterEnding(last_encounter)     
    if last_encounter == "Encounter-Only" then
        Overworld.CutsceneObj.startSpecialCustscene[2] = true
    end
end


    Once Overworld.CutsceneObj.startSpecialCustscene[2] has been set, the respective cutscene within the Cutscenes script will start.
