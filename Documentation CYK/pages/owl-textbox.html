Textbox Functions & Properties


    In-line Functions



    TalkingSprite( ogmoObjectID, isTalking)

    TalkingSprite is NOT available on the optimized example!

    1. The function scans the Triggers layer ONLY, and finds the trigger whose ID matches the provided ogmoObjectID.
      You can view each trigger's ID within the Ogmo editor.

    2. If TalkingSprite is set true, the sprite is set to its "talking" animation.
      If false, the sprite is set to its "quiet" animation.

    3. You must call this function both at the start of your dialogue, as well as at the end, before the last character.
      TalkingSprite + the Textbox automatically ensure the given sprite returns to its "quiet" animation if the player skips the text with the Menu key.



    Here is an example on how to build a sprite compatible with the TalkingSprite function.

    This code must be run within Overworld/RoomHandler.

    function self.OnRoomSetup(roomName)
    if roomName == "PartyChangingRoom" then
        local lancer = self.FindObjectInRoom("Triggers", 0)

        lancer["animPrefix"] =  "Overworld/Lancer" -- This prefix is used to find the frames of each animation!
        lancer["quiet"]      =  { 0 }
        lancer["talking"]    =  { 1, 2, 3, 4, 5, 6, 7, 8, 9, 9, 9, 9 }

        lancer["sprite"]     = CreateSprite("Overworld/Lancer/0", "OWEntities")
        lancer["animspeed"]  = 1/9

        lancer["sprite"].SetPivot(0.5, .5)
        lancer["sprite"].MoveTo(lancer.x+25, lancer.y)
        lancer["sprite"].SetParent(lancer.hitbox)  -- Do this, and you won't have to delete this sprite manually once the room is destroyed!

        -- If you do wish to delete it manually, though, add it to this queue:
        table.insert(self.spriteTrashQueue, lancer["sprite"])
    end
end



    SetMugshot( faceSpriteString) The +Overworld's counterpart to 's Overworld mugshot() function.

    A faceSpriteString string is structured like so:

    Speaker.Face

    1. Speaker: The name of the Folder within Sprites/Overworld/FaceSprite where Face is located.

    2. Face: The sprite to be displayed. Recommended resolution: 112x112

    All Speaker's must have a valid SpeakerInfo counterpart within Overworld.TextBox.SpeakerInfo.

    Read the following section to learn how to set one up.

    RoomHandler Functions

    These functions can be called within your Overworld/RoomHandler script.

    If calling from the Overworld/RoomHandler script, you may call these functions as: self.FunctionName().
    Otherwise, if calling from your Encounter script or Overworld/Cutscenes, you must call them as: Overworld.FunctionName()



    self.Textbox.CreateTextbox( dialogue, isAbove = false,
    (Textbox.closingModeEnum) closingMode = Textbox.closingModeEnum.toControllable )[E/O]

    More often than not, you won't be calling this function manually.


    1. dialogue: The dialogue's to be displayed on the Textbox. You obtain said by calling: self.Dialogues.getDialogue( self.roomName, nextDialogueTag)
    2. isAbove: If true, the Textbox is shown closest to the top margin of the screen. Otherwise, it's shown closest to the bottom margin (probably what you're used to).

    3. closingMode: An enum that determines a specific action to be performed by this Textbox once it's done being read.
      1. toNothing: Performs nothing. Default in cutscenes.
      2. toControllable: Makes the player's Avatar controllable. Default value.

    Notice: Overworld/Cutscenes has a built-in shorthand for this function: self.Textbox( nextDialogueTag), paired by the self.isTextboxAbove property.



    self.HandleChoice( pickedDialogueTag) [O] Called when the player makes a choice at the end of the Textbox, with the respective dialogue's tag. For more information on how Choices are structured, visit the Loading Dialogues page.

    This is a customizable function: meaning, rather than calling it, you write inside it, modifying the function's contents.



    Here is a simplified example: 

    function self.HandleChoice(pick)
    -- By default, the "close" tag simply closes the textbox,
    -- then returns control to the Player if a cutscene isn't running.
    if pick == "close" then
        self.canControl = ( self.currentCutscene == nil )
        return
    end
    
    -- Here is some custom behaviour...
    if pick == "scene3-redo" then
        Overworld.StartBattleIntro("Encounter-Only", false)
        Overworld.CutsceneObj.currentCutscene()
        State("INTRO")
        return -- Ensure you've called "return" at the end of your code block.
    elseif pick == "scene3-done" then
        State("DONE")
        return -- Ensure you've called "return".
    end


    -- If "return" wasn't called beforehand, then the default behaviour of
    -- calling a new textbox takes place.
    self.TextBox.CreateTextbox( Overworld.Dialogues.getDialogue(self.roomName, pick) )
end
