OWL Docs - Assets (aka "Sprites" in Ogmo)



    Assets must be placed into the respective Assets layer via the Ogmo Editor



    Seriously, watch it. That's the most common cause for bugs I found on my work.

    Properties






    rescale = false If true, scales up this sprite by 2.


    hasYSort If true, adds this sprite into the Overworld.overworldYSortQueue.

    By implication, this binds your sprite into the OWEntities Sprite Layer.


    spritePath The path pointing towards the Sprite to be used.

    Worry not about the "../../". The library trims it out.
