How to read this documentation



    Reading Functions

    Let's consider the following function:


    Screen.DispImg( path, posX, posY, ID = 0 )

    • The words separated via comas with no spaces in between are known as Arguments. When calling the function, you must fill out each of them with a value that respects its given type.
    • The highlighted words placed before the argument (such as , ...) indicate the argument's type. When calling the function, each filled out value must respect its given type. You'll encounter errors otherwise.
    • Consider the ID argument. It's been written as ID = 0, hasn't it? What this means is the ID argument has been given a default value. If you don't fill out said argument when calling the function, its default value will be used instead.

    For instance, you can call the following function as such without prompting any errors:

    Screen.DispImg("poseur", 320, 480) The ID argument will simply be assumed as 0 if you do so.

    Considering a term's scope (E, M, P and W)

    All over this documentation, you will find...

    isCYF [E/M/P/W] [E], [M], [P], [W], or a mix of the four.

    These simply mean that the relevant variable, function or object is only accessible from:

    • [E]: Encounter scripts.
    • [M]: Monster scripts.
    • [P]: Player scripts.
    • [W]: Wave scripts.

    If something's tagged as pertaining an Encounter script, it functionally means it can accessed from anywhere within your mod. For instance, any library files!
