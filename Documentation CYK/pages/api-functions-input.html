    Do not rely on the Input object to replace proper UI controls. Changing the game state in the UI based on input will likely cause a fair share of issues and is not supported at this moment (but feel free to see what does and doesn't work).

    Input.GetKey( keyname) Gets the "state" of the given key. Each available keyname returns a given number depending on its current state.

    • 0 if not pressed.
    • 1 during the first frame the key is pressed.
    • 2 while being held down.
    • -1 during the first frame the key is released.
    You can simply check if a key's value is greater than 0 to tell if it's active at all.

    Be sure to check the Key List to further understand keynames.


    Input.MousePosX (readonly) - Returns the X position of the mouse relative to the bottom left corner of the screen. (from 0 to 640)


    Input.MousePosY (readonly) - Returns the Y position of the mouse relative to the bottom left corner of the screen. (from 0 to 480)


    Input.IsMouseInWindow (readonly) - Returns true if the mouse is in the window, false otherwise.
