Textboxes - Loading Dialogues



    You will need to create a room in your Encounter's EncounterStarting() call.

    Unless you're using the optimized example, this is non-negotiable.


    Each room within Overworld/Rooms has a JSON Dialogue counterpart, located within Overworld/Dialogues.
    Both files must share the same name.

    Notice: If (and only if) using the optimized example, rooms will be ignored and, instead, all dialogues will always be loaded from within the Overworld/AllDialogues.json file.


    Below is the empty room's JSON file as an example. 

    {
    "scene1": ["[func:SetMugshot,Gentle.Pout]I have a mouth I just don't feel like screaming.",  
    [   [],                   -- "Up" choice.
        ["One", "scene2"],  -- "Right" choice.
        [],                   -- "Down" choice.
        ["Two", "close"] ]  -- "Left" choice.
    ],

    "scene2": ["[func:SetMugshot,Susie.Pout]Do you care? No? Okay then."] 
}

    Writting Dialogues



    Here's a brief summary of the data you've read above:

    • Dialogues are stored as a .

    • If you wish to add a choice, you must make the last value of the Dialogue's , a with 4 items (rather than a ...)

    • The Choice's items are ordered clockwise to how they'll be presented on the Textbox:
      1. is UP.
      2. is RIGHT.
      3. is DOWN.
      4. is LEFT.

    • Each Choice is a with two items:
      1. is the displayed on the Textbox.
      2. is the new dialogue's ID tag().
        If the player selects this choice, then this tag will then be passed down onto the Overworld's HandleChoice function.
