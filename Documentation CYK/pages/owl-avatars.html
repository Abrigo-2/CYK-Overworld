- Avatars



    Avatars serve a two-fold purpose:

    • They are the sprites which the player uses to navigate and interact with the Overworld interface.
    • They retain player HP after a battle.
    • Optionally, they determine your battle's party members. (Check the Encounter Only demo!)


    In this documentation, "active party" refers to the Overworld.party avatars.

    Meanwhile, "player's avatar" refers to the first Avatar within the Overworld.party's .
    This is the Avatar the player controls in order to navigate the Overworld. The other party members simply follow them around.



    Creating and Loading Avatars



    Avatar scripts are loaded similarly to Entities (actually, identically: they even use the same function).

    First off,

    Your script must be located within Lua/Overworld/Avatars.



    Once the mod is intialized, all Avatar scripts within that folder are then immediately loaded into the Overworld.allAvatars's avatars.

    Avatar Properties



    hp Moonlights as this Avatar's Max HP during initialization.
    Afterwards, this simply stores the Avatar's HP after a Battle.


    battleName The name of this Avatar's Player Entity script within Lua/Players.

    Only used by the Encounter Only demo!



    hitbox The Avatar's collision box. This is only taken into account for the player's avatar, which is only checked against the "solid" Tileset Layer.
    For checking collissions against Triggers, the player's avatar exact coordinate point is used.

    This has the following properties:

    hitbox.startX, hitbox.startY The box's offset from the Avatar's sprite pivot.
    Every Avatar's sprite pivot is set to x=0.5, y=0

    hitbox.width, hitbox.height The box's size.


    direction The Avatar's current direction they are looking at (or walking towards.)

    Avatar directions are ordered clockwise, meaning:

    1. is UP.
    2. is RIGHT.
    3. is DOWN.
    4. is LEFT.



    sprite sprite Same as 's Player Entities.


    animations animations Same as 's Player Entities.


    animsFolder The folder within Sprites/Overworld where this Avatars' animations folders are located.


    Setting the active party's Avatars

    • You may have as many Avatars on your active party as you'd like.

    • Out of all the Avatars in your active party, only the the first 3 are used during battles.

    • By default, only 4 party members are saved at a time. You can change this.



    Setting the Avatars within your Encounter's EncounterStarting() Event function. Simply do: 

    function EncounterStarting()
    Overworld.party[1] = Overworld.allAvatars["OWKris"]
    Overworld.party[2] = Overworld.allAvatars["OWRalsei"]
    ...
end


    Setting the Avatars during runtime. Check out the following function:

    Overworld.SwapPartyMember( placeInParty, avatarName) Meant to be used during runtime.
    If on your Encounter's EncounterStarting() function, you can simply do: Overworld.party[i] = "AvatarName".

    Swaps the avatar avatarName with the party member at the given placeInParty. (Inside the Overworld.party's ). You can even swap with "nil"!
    This function will automatically set their position, direction and sprites up for you.
    These will be the same as active player's position and direction unless you set them afterwards.

    An Avatar's name is the same name of their respective Overworld/Avatars Lua script!


    Cutscene Functions



    The following are functions to aid you in manipulating Avatars before, during, or after a given Cutscene.

    Remember: When calling these within Lua/Overworld/Cutscenes, you cannot use self.FunctionName()!



    Overworld.StopPlayer() [O] Stops all Avatars' "Walk" animation (if playing), and sets Overworld.canControl to false.


    Overworld.ToggleAvatars( alpha) [O] Sets the alpha (aka, opacity (aka, transparency)) of all Avatar sprites to the given alpha.
    This is the only way to hide a sprite in , far as I know.


    Overworld.PlayerLookAt(avatar avatarObj, direction, dontOverrideDirection = false) [O] Sets the avatarObj's sprite animation to "Idle", making them "look at" the given direction.

    Set dontOverrideDirection true when calling this right before Overworld.Movement.MakeFollowPathTowardPlayer for this avatar in order to avoid flickering in the "Walk" animation.
    In other words: Set true for party members (that aren't the player's avatar) at the end of a cutscene!





    Overworld.party[i].posX, Overworld.party[i].posY Same as 's Player Entities.


    Overworld.party[i].MoveToAbs( absX, absY) Same as any sprite or entity.


    Overworld.Movement.MakeFollowPathTowardPlayer( partyMemberID, fromAbsX, fromAbsY, toAbsX, toAbsY, direction) Forges a "path" for this avatar at Overworld.party[partyMemberID] to follow whilst displaying the walking animation for the given direction, before becoming "glued" to the player's avatar breadcrumbs position & direction.

    Yes, it is mostly a required function to cope with the quirks of the engine, but the point is, you'll want to call it for party members at the end of a cutscene.


    Here is an example for how you would wrap-up a cutscene: 

    ...
    -- Of course, we asume self.actor[1] & self.actor[2] were defined in the prior cutscene
    Overworld.party[1].MoveToAbs( self.actor[1].absx, self.actor[1].absy )  -- Position.
    Overworld.PlayerLookAt(Overworld.party[1],  3) -- "Idle" direction.

    Overworld.party[2].MoveToAbs( self.actor[2].absx, self.actor[2].absy )
    Overworld.PlayerLookAt(Overworld.party[2],  2, true)
    
    Overworld.Movement.MakeFollowPathTowardPlayer(2,  -- Party member ID
    self.actor[2].absx, self.actor[2].absy,  -- "From" position
    self.actor[1].absx, self.actor[1].absy,  -- "Towards" position
    2) -- Direction the party member will look at during the walk.


    -- We don't need the actors anymore.
    self.actor[1].Remove()
    self.actor[2].Remove()
    self.actor = {}

    self.ToggleAvatars(1)  -- Make the party members visible again.  
    
    self.currentCutscene   = nil -- Stops cutscene playback.

    -- Then return player control.
    Overworld.cameraFollowPlayer = true
    Overworld.canControl = true
    ...
