    * + Overworld's feature Changelog

    Entities.

    • Damage formula has been made accurate to Deltarune's.
    • TP is stored by its Net value (0 to 250), rather than its % value.
    • Speech Bubbles can now be added to Players. This meant some changes were required on how speech bubble dialogues are stored.
    • The Update() function will be run for each entity even if isactive has been set false.
    Read the respective Entities section for details.

    CYKCore.
    • "Encounter" file funcionality has been taken over by "Battle" files.
      Battle files store the data and functions for each Encounter, which are then loaded during runtime. This allows mods to have more than one Encounter in them, without needing to end the mod!
    • An additional SETUP game State was added to the State function. It's now required in order to build the Encounter's battle UI and entities.
    • HandleCustomSpell has been broken up into two functions: PrepareCustomSpell and HandleCustomSpell.
      Same with HandleItem: HandleItemDialogue and HandleItem.

      One is called at the start of the Player's turn, the other is called once that Player's animation reaches its actionFrame.

    • Related to the code above, HandleSpare's BattleDialog text is stored inside CYKCore. HandleSpare is called once the Player's SPARE animation reaches its actionFrame.
    • BattleDialog's Flavor Text and Monster Speech Bubbles can now be skipped by holding the C key.
    Read the respective Entities section for details.

    Wave Scripts.
    • The CreateProjectile methods will initially look for sprites at the Sprites/Bullets/ directory.
    • A CreateArenaSprite method was added, inteded for making sprites during Waves that'll then be automatically freed once the Wave ends.
    Text Objects.
    • Both the Battle and +Overworld textboxes have a white gradient on top of the displayed font. Setting a color via [color:rrggbb] on your dialogues (or changing the font) will make this noticeable.
    Polish.
    • A certain StartSecondaryAnimation function was added to CYKCore's Animation script. It takes care of starting up polish effects, such as the Heal green sparkles, the perfect-hit stars (since the old implementation was somewhat buggy), and the Spare fail/Pacify fail animations.
      I never added the Pacify animation...
    • The Screenshake and Entity's hurt animations were made more accurate with Deltarune's.
    • Susie's Rude Buster animation was made more accurate.
